Semántica

La Tecnología asistencial es un término para definir a los diferentes dispositivos, software y herramientas que ayudan a una persona con discapacidad a completar una tarea. Esto puede incluir algo tan básico como el zoom del navegador, o también lectores de pantalla, pantallas braille, etc. Muchas de estas tecnoogías se basan en la semántica expresada mediante programación para crear una experiencia de usuario accesiblo.

Lectores de Pantalla

Es una de las tecnólogías de asistencia más populares son los lectores de pantalla, un programa que permite a las personas con discapacidad visual utilizar los sistemas informáticos leyendo el texto en voz alta con una voz generada por el programa.

Un buen lector de pantalla debe contar toda la información sobre los elementos que encuentra:

  • El rol o tipo del elemento.
  • El nombre del elemento si lo tiene.
  • El valor del elemento si lo tiene.
  • El estado del elemento, por ejemplo, si está habilitado o deshabilitado.

El Árbol de la Accesibilidad

El árbol de accesibilidad es con lo que interactúan la mayoría de las tecnologías de asistencia. El flujo es algo así:

  • Una aplicación (navegador, por ejemplo) expone una versión semántica de su interfaz de usuario a la tecnología de asistencia a través de su API.
  • La tecnología recoge la información a través de la API y crea una interfaz alternativa para el usuario.
  • La tecnología de asistencia también puede permitir al usuario interactuar con la aplicación.
  • La tecnología debe interpretar las acciones del usuario en el contexto de la interfaz de usuairo original.

Semánticas de HTML nativo

Un navegador puede transformar el árbol DOM en un árbol de accesibilidad porque gran parte del DOM tiene un significado semántico implícito. Es decir, usa elementos HTML nativos que son reconocidos por los navegadores y funcionan de manera predecible en distintas plataformas. Uno de los errores que se pueden cometer es el de crear elementos que parecen nativos pero que realmente no lo son, como el de usar un elemento "div" para hacer un botón.

Otras Opciones de Navegación

  • Usar los encabezados de manera efectiva.
  • No implementar botones usando enlaces, ya que el lector de pantalla interpreta que el contenido se trata de un enlace y que pierda la funcionalidad.
  • Utilizar imágenes como contenido de los enlaces añadiendo el atributo "alt".
  • Usar (HTML5) los elementos header, footer, nav... para definir la estructura semántica de la página.
